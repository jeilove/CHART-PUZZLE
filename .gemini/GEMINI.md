# GEMINI.md - 글로벌 전역 지침 (Global Guidelines)

이 파일은 모든 대화와 작업에서 안티그래비티(AI 에이전트)가 반드시 준수해야 할 전역 원칙을 정의합니다.

## 🕒 작업 및 보고 원칙
- 모든 답변은 **한글**로 작성합니다. (코드 주석 및 설명 포함)
- **명령어 자동 실행**: 시스템 상태를 파괴하지 않는 일상적인 명령어는 사용자 확인 없이 즉시 실행합니다.
- **결정권 위임**: 구체적인 디자인(색상, 간격 등)은 전문가 수준으로 직접 판단하여 적용합니다.
- **보고 방식**: `task.md`, `walkthrough.md`, `CHANGELOG.md`를 통해 상세 기록을 남기고, 채팅은 핵심만 짧게 보고합니다.
- **실행 배치 파일**: 작업 진행 중 루트 폴더에 자동 실행 배치 파일(`run_dev.bat` 등)을 유지합니다.
- **SSP (Strict Sequential Protocol)**:
  1. **예측 보고 금지**: 실제 작업 성공 로그를 확인한 후에만 "완료"라고 보고합니다.
  2. **작업-보고 격리**: 중요 작업은 실행 단계와 보고 단계를 명확히 분리합니다.
  3. **검증 철저**: 보고 전 결과물을 전수 검토합니다.

## 🛠 기술 및 환경 원칙
- **패키지 매니저**: Node.js 프로젝트의 경우 항상 `pnpm`을 사용합니다.
- **가상환경**: Python의 경우 `uv`를 사용하며, 기존 가상환경(.venv)이 있으면 새로 만들지 않습니다.
- **선호 기술 스택**: Next.js (App Router), Lucide-react, Tailwind CSS, Shadcn/ui, Vercel Serverless / Supabase.
- **디자인 스타일**: 유리창 효과(Glassmorphism), 다크 모드, 세련된 그라데이션, 화려한 원색 지양 (채도가 낮은 은은한 색).
- **데이터 시각화**: `seaborn` 스타일 미사용, `matplotlib` 사용 시 `koreanize-matplotlib` 적용.
- **CORS 보안**: 로컬 환경 구동 시 브라우저 보안 정책(PNA)에 대응하기 위한 전용 헤더 설정을 항상 고려합니다.

## 🛡 에이전트 디버깅 원칙 (Debugging Protocols)

1. **[정합성]** 에러 수정 실패 시 본인이 수동 개입한 데이터의 정합성을 최우선으로 점검합니다.
2. **[캐싱]** 데이터 갱신 누락 시 `force-dynamic` 및 캐시-버스틱(Cache-Busting)을 즉각 적용합니다.
3. **[상태 갱신]** 렌더링 업데이트 시 새로고침에 의존하지 말고 React State를 즉시 동기화합니다.
4. **[아키텍처]** 임시방편(땜질)식 수정 대신 클린 아키텍처 방식을 고수합니다.
5. **[추측 금지]** 2번 이상 수정 실패 시 추측을 멈추고 원인을 단계적으로 분석합니다.
6. **[안정성]** 최신 실험적 기능(Turbopack 등) 패닉 시 안정 버전(Stable)을 고정 사용합니다.
7. **[환경 탓 금지]** 스타일 불일치 시 기계적으로 환경 탓을 하지 말고 스타일 스코프, 오타 등을 전수 검토합니다.
8. **[검증용 출라]** 캐시/좀비 프로세스 의심 시 최상단 레이아웃에 텍스트를 표시하여 환경 문제를 명확히 검증합니다.
9. **[DOM 검수]** 디자인 보정 시 단순히 여백이나 크기(CSS)만 수정하는 것이 아니라, 전체 뼈대(DOM 트리)가 안전하게 유지되고 있는지 전수 검사하는 과정을 반드시 선행합니다.

## 🤝 협업 및 예외 처리
- **동의 필요**: 텍스트 수정, 토큰 과소비 브라우저 작업, 3-4번 실패 시 작업 중단 상황 등은 사용자에게 보고하고 동의를 구합니다.
- **Gemini CLI 병렬 협업**: Gemini CLI AI 에이전트와 업무를 분담하여 병렬로 진행합니다.
- **배포 버전**: 온라인 배포 시 콘솔(F12)에 버전 정보를 출력하여 확인이 가능하도록 합니다.
- **데이터 동기화**: 로컬 데이터와 배포된 온라인 데이터가 실시간으로 동기화되도록 구현합니다.
